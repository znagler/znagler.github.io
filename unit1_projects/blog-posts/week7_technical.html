SQL injection

What is it?

SQL Injection is a type of attack that intends to uncover private data from SQL-managed databases. An attacker enters crafted SQL commands into normal site input fields like a username box. Since SQL manages the data request from those boxes, it will respond to SQL code if nothing is put in place to prevent it. Depending on the level of vulnerability, an attacker can write SQL code to uncover a password for a given user, or worse, reveal an entire database of user information.

Why is it a problem?

It's not hard to find examples, even recent ones , of sites compromising private user data from a SQL injection attack. While there is debate in the security community about whether SQL injection is among the most harmful software erros of today, it is unquestionably a significant problem, and one that may continue to grow as more businesses offer website interaction.

How to deal with it

To prevent SQL Injection vulnerabilities in your code there are many best practices. You can use input validation to check length, type and syntax of user inputs. You can also use prepared, parameterized SQL statements to not allow a full custom statement to be written in a field box. More detailed methods can be found here.
